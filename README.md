# Math-168-Final-Project
This repository contains all data, code, and conceptual procedure planning aspects of our project for Math  168, which focuses on protraying basketball games as a series of networks (one weighted, undirected schedule network, and another weighted, directed game results network, where loser $i$ points to winner $j$. The general goals and purpose of this project can be found in the attached paper "Bridge Redundancy and Spectral Comparability in Basketball Networks". 
## Data
This repository has .csv files taken from the Kaggle dataset "March Machine Learning Mania 2026" (https://www.kaggle.com/competitions/march-machine-learning-mania-2026/data), including regular season and tournament game win-loss information, team identification information, and conference identification information. It also contains .csv files created by Jared Gode using data from the college basketball Sports Reference page (https://www.sports-reference.com/cbb/), used to create early adjacency matrices for data exploration and to gain an understanding of the usage of the NetworkX package
## Code
The .ipynb files in this notebook work through the following: (1) Early data exploration and NetworkX package exploration can be found in WebScrapeTrialMath168.ipynb; (2) Initial implementation of the Kaggle data and early centrality analyses, credibility approximation calculations, and latent strength calculations can be found in CBB_Network_Rankings.ipynb; (3) Approaching Kaggle dataset from a different angle, focusing on the community structure/modularity of the networks, the bridges between these communities/groups, and the embeddedness of nodes or teams in the general schedule network in ncaa_network_project_null_models_v4-2.ipynb; (4) Unifying the ideas discussed in notebooks 2 and 3, we build on the network specific concepts developed in notebook 3 and use them to make the models introduced by notebook 2 more robust, showing how team strength predictions can potentially be improved by considering conferences and their modularity or bridge connections to other conferences in ncaa_network_project_null_models_v8_tournament_extension.ipynb.
## Process
PDF files found in this repository can be read to follow the brainstorming process, aka how we approached the matter of representing college basketball seasons as networks, but much of the information found in these files is redundant or less robust than similar information found in the python notebooks explained above.
## Replication
Basic formation of the graphs or networks can be easily mirrored following the early steps of each python notebook, NetworkX implementation via the Kaggle data is straightforward, and subsequent calculation/modeling/visualization using the networks has the procedure explained effectively in each notebook, and additional information on the idea behind these calculations or models can be found in the process pdf documents.
